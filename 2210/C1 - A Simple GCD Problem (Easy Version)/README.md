The difference between the versions is that in this version, $$$1 \leq n \leq 2 \cdot 10^{5}$$$ and $$$b_i = a_i$$$ for $$$1 \leq i \leq n$$$. Note that a solution for one version does not necessarily solve the other version. You can hack only if you solved all versions of this problem.</span> </p><p>You are given two arrays $$$a$$$ and $$$b$$$ of length $$$n$$$.</p><p>For <span class="tex-font-style-bf">each index</span> $$$i$$$ ($$$1 \le i \le n$$$) of array $$$a$$$, you can perform the following operation <span class="tex-font-style-bf">at most once</span>: </p><ul> <li> choose an arbitrary integer $$$m$$$ ($$$\mathbf{m \neq a_i}$$$) such that $$$1 \leq m \le b_i$$$, and set $$$a_i := m$$$. </li></ul><p>Let the array after performing all the operations be $$$a'$$$. You can only perform operations in such a way that the following condition holds: </p><ul> <li> for all $$$1\leq l \lt r\leq n$$$, $$$\operatorname{gcd}(a_l,a_{l+1},\ldots,{a_r})=\operatorname{gcd}(a'_l,a'_{l+1},\ldots,a'_r).$$$ </li></ul><p>Here, $$$\gcd(x, y)$$$ denotes the <a href="https://en.wikipedia.org/wiki/Greatest_common_divisor">greatest common divisor (GCD)</a> of integers $$$x$$$ and $$$y$$$.  </p><p> You have to determine the <span class="tex-font-style-bf">maximum</span> number of operations that can be performed while ensuring that the condition remains satisfied.</p></div><div class="input-specification"><div class="section-title">Input</div><p>Each test contains multiple test cases. each test case contains $$$n$$$ space-separated integers $$$a_1, a_2, \ldots, a_n$$$ ($$$1 \leq a_i \leq 10^9$$$).</p><p>The next line of each test case contains $$$n$$$ space-separated integers $$$b_1, b_2, \ldots, b_n$$$ ($$$\color{red}{b_i = a_i}$$$).</p><p>It is guaranteed that the sum of $$$n$$$ over all test cases does not exceed $$$2\cdot10^5$$$. </p></div><div

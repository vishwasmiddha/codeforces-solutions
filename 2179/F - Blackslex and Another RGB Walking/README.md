## Problem Statement

<div class="header"><div class="title">F. Blackslex and Another RGB Walking</div><div class="time-limit"><div class="property-title">time limit per test</div>5 seconds</div><div class="memory-limit"><div class="property-title">memory limit per test</div>256 megabytes</div><div class="input-file input-standard"><div class="property-title">input</div>standard input</div><div class="output-file output-standard"><div class="property-title">output</div>standard output</div></div><div><p><span class="tex-font-style-it">This is a run-twice (communication) problem.</span></p><p>There are two players: Player A (Agent) and Player B (Blackslex). The jury will first interact with player A. After player A ends their interaction, the jury will interact with player B. Note that player A and player B may not directly pass information to each other; both players are only able to send information or receive information from the jury, but they may agree on the strategy they will use to communicate.</p><p>The Penguin Republic is a <a href="https://en.wikipedia.org/wiki/Bipartite_graph"><span class="tex-font-style-bf">bipartite</span></a> connected undirected graph $$$G$$$ with $$$n$$$ vertices and $$$m$$$ edges. Blackslex is going to conduct forbidden field research at vertex $$$1$$$. Due to travel restrictions, he will be dropped off at an unknown vertex $$$v$$$ ($$$2 \leq v \leq n$$$). He must get to vertex $$$1$$$ while having no information on the graph.</p><p>For his journey, he has bribed a penguin agent and agreed to some communication strategy using the following method; the agent will discreetly mark each vertex in one of the three colors: red, green, or blue. From Blackslex's perspective, he will see only the color $$$c_i$$$ of each neighbor $$$u_i$$$ ($$$1 \leq i \leq d(v)$$$$$$^{\text{∗}}$$$) of $$$v$$$. He must choose some $$$j$$$ ($$$1 \leq j \leq d(v)$$$) and move to vertex $$$u_j$$$ such that he is closer to vertex $$$1$$$.</p><p>Note that the neighbors are arbitrarily ordered. He sees only the colors of the neighboring vertices, and not the vertex that he is on. Additionally, he does not know the index of the vertex he's on, the neighboring vertices, or any other vertex.</p><p>Your task is to implement the strategy for both the agent and Blackslex. For the agent, you must color each vertex in one of the three colors. For Blackslex, you are given $$$q$$$ queries. In each query, you are dropped off at an arbitrary and unknown vertex $$$v$$$ and given the color of the neighboring vertices. You must determine a vertex to go to such that you are closer to vertex $$$1$$$.</p><div class="statement-footnote"><p>$$$^{\text{∗}}$$$The number of neighbors of vertex $$$v$$$.</p></div></div><div class="input-specification"><div class="section-title">Input</div><p>Your code will be ran exactly two times on each test. On the first run, you will be Player A (Agent), and on the second Player B (Blackslex).</p><p><span class="tex-font-style-bf">First Run Input</span></p><p>The first line of the input contains the string <span class="tex-font-style-tt">first</span>. The purpose of this is so your program recognizes that this is its first run, and it should act as Player A.</p><p>Each test contains multiple test cases. The first line contains the number of test cases $$$t$$$ ($$$1 \leq t \leq 10^4$$$). The description of the test cases follows.</p><p>The first line of each test case contains two integers $$$n$$$ and $$$m$$$ ($$$2 \leq n \leq 10^5$$$, $$$n-1 \leq m \leq 10^5$$$) — the number of vertices and edges respectively.</p><p>The following $$$m$$$ lines contain information about the edges. The $$$i$$$-th ($$$1 \leq i \leq m$$$) line has two integers $$$a_i$$$ and $$$b_i$$$. ($$$1 \leq a_i, b_i \leq n$$$, $$$a_i \neq b_i$$$) — vertex $$$a_i$$$ is connected to vertex $$$b_i$$$ by edge $$$i$$$.</p><p>It is guaranteed that: </p><ul> <li> The sum of $$$n$$$ and the sum of $$$m$$$ does not exceed $$$10^5$$$ over all test cases. </li><li> The graph in each test case is bipartite and connected. It has no duplicate edges and no self-loops. </li></ul><p><span class="tex-font-style-bf">Second Run Input</span></p><p>The first line of the input contains the string <span class="tex-font-style-tt">second</span>. The purpose of this is so your program recognizes that this is its second run, and it should act as Player B.</p><p>Each test contains multiple test cases. The first line contains the number of test cases $$$t$$$ ($$$1 \leq t \leq 10^4$$$) — the same value of $$$t$$$ in the first run. The description of the test cases follows.</p><p>The first line of each test case contains one integer $$$q$$$ ($$$1 \leq q \leq 10^5$$$) — the number of queries in this test case.</p><p>The first line of each query contains one integer $$$d(v)$$$ ($$$1 \leq d(v) \leq 10^5$$$) — the number of neighbors of the vertex $$$v$$$ that Blackslex is currently on.</p><p>The next line of each query contains a string $$$c$$$ of length $$$d(v)$$$ — the $$$i$$$-th ($$$1 \leq i \leq d(v)$$$) character of the string is the color of the neighboring vertex $$$u_i$$$. The characters in the string are <span class="tex-font-style-tt">r</span>, <span class="tex-font-style-tt">g</span>, or <span class="tex-font-style-tt">b</span> representing red, green, or blue.</p><p>It is guaranteed that: </p><ul> <li> The sum of $$$q$$$ does not exceed $$$10^5$$$ over all queries in all test cases. </li><li> The sum of $$$d(v)$$$ does not exceed $$$2 \cdot 10^5$$$ over all queries in all test cases. </li><li> $$$v \neq 1$$$ </li></ul><p>The input of the second run is not adaptive. In other words, the input of the second run will not change in different runs.</p><p><span class="tex-font-style-bf">Hacks</span></p><p>To make hacks, use the following format:</p><p>The first line contains one integer $$$t$$$ ($$$1 \leq t \leq 10^4$$$) — the number of test cases.</p><p>The description of the test cases for the first run follows.</p><p>The first line of each test case contains two integers $$$n$$$ and $$$m$$$ ($$$2 \leq n \leq 10^5$$$, $$$n-1 \leq m \leq 10^5$$$) — the number of vertices and edges respectively.</p><p>The following $$$m$$$ lines contain information about the edges. The $$$i$$$-th ($$$1 \leq i \leq m$$$) line has two integers $$$a_i$$$ and $$$b_i$$$. ($$$1 \leq a_i, b_i \leq n$$$, $$$a_i \neq b_i$$$) — vertex $$$a_i$$$ is connected to vertex $$$b_i$$$ by edge $$$i$$$.</p><p>After that, the description of the test cases for the second run follows.</p><p>The first line of each test case contains one integer $$$q$$$ ($$$1 \leq q \leq 10^5$$$) — the number of queries in this test case.</p><p>The first line of each query contains one integer $$$v$$$ ($$$2 \leq v \leq n$$$) — the vertex the Blackslex is dropped off.</p><p>The second line of each query contains $$$d(v)$$$ integers $$$p_1, p_2, \ldots, p_{d(v)}$$$ ($$$1 \leq p_i \leq d(v)$$$, each number in $$$p$$$ is distinct) — the ordering of the neighbor is as follows; let $$$q_1  \lt  q_2  \lt  \ldots  \lt  q_{d(v)}$$$ be the neighbors of $$$v$$$, then the input order of the neighbor is $$$u_i = q_{p_i}$$$.</p><p>It must hold that: </p><ul> <li> The sum of $$$n$$$ and the sum of $$$m$$$ does not exceed $$$10^5$$$ over all test cases in the first run. </li><li> The graph in each test case is bipartite and connected. It has no duplicate edges and no self-loops. </li><li> The sum of $$$q$$$ does not exceed $$$10^5$$$ over all queries in all test cases. </li><li> The sum of $$$d(v)$$$ does not exceed $$$2 \cdot 10^5$$$ over all test cases in the second run. </li></ul></div><div class="output-specification"><div class="section-title">Output</div><p>For the first run, for each test case, output a single string $$$s$$$ of length $$$n$$$ — $$$s_i$$$ ($$$1 \leq i \leq n$$$) is the color of the $$$i$$$-th vertex, painted by the agent. The characters in the string are <span class="tex-font-style-tt">r</span>, <span class="tex-font-style-tt">g</span>, or <span class="tex-font-style-tt">b</span> representing red, green, or blue.</p><p>For the second run, for each query in each test case, output a single integer $$$j$$$ ($$$1 \leq j \leq d(v)$$$) — $$$u_j$$$ is the neighboring vertex that Blackslex will go to next.</p></div><div class="sample-tests"><div class="section-title">Examples</div><div class="sample-test"><div class="input"><div class="title">Input<div title="Copy" data-clipboard-target="#id007848730903884488" id="id0005581793540818314" class="input-output-copier">Copy</div></div><pre id="id007848730903884488">first
